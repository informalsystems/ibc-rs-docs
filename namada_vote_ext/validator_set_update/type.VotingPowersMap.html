fn get_abi_encoded(&self) -> (Vec<Token>, Vec<Token>)

Returns the list of Ethereum validator hot and cold addresses and their respective voting powers (in this order), with an Ethereum ABI compatible encoding. Implementations of this method must be deterministic based on self. In addition, the returned Vecs must be sorted in descending order by voting power, as this is more efficient to deal with on the Ethereum side when working out if there is enough voting power for a given validator set update.
